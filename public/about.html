About This Page

General Description

Simply Art Gallery is a full-stack web application that allows users to browse, search, and manage artworks. Users can add artworks to their cart, complete a checkout process to add items to their personal collection, favourite artworks for quick access later, and edit or remove items from their collection. The page content is dynamically retrieved using AJAX calls to a Node.js server, and updates happen without needing full page reloads.

Technologies Used

  • Frontend: HTML, CSS, JavaScript, and FontAwesome icons
  • Backend: Node.js with Express.js for the server
  • Database: MongoDB (In-memory MongoDB server)
  • APIs: Custom RESTful API endpoints for fetching, creating, updating, and deleting artworks
  • Local Storage: Used to temporarily save cart, favourites, and collection data on the browser

Weaknesses of the Application

  • Data persistence is currently based on Local Storage, so collections and favourites are lost if the browser data is cleared.
  • There is no user authentication — anyone accessing the site shares the same local storage data.
  • Using an in-memory database means all artwork data would reset if the server restarts.
  • Limited backend validation — the server trusts data sent by the client without deep verification.

Alternative Approaches

  • Use a real hosted MongoDB database (e.g., MongoDB Atlas) to store user data persistently across sessions.
  • Implement authentication and account management so users have private collections and carts.
  • Replace Local Storage with server-side session management or JWT-based authentication.
  • Deploy the backend and frontend using a cloud platform like Vercel, Heroku, or Railway for live access.
